# 工具版本与发布门禁

## 版本真源

- 统一版本注册表：`templates/versions/registry.json`
- 路由注册表：`config/seedance_routing_registry.yaml`
- 工具导入镜像：`templates/*_tool_v*.import.json`

## 发布门禁（必须同时满足）

1. 工具源码与 import 镜像一致：
   - `python scripts/seedance/sync_template_imports.py --check`
2. 路由/技能/提示词完整性检查通过：
   - `python scripts/seedance/check_templates_integrity.py`
3. 自动化测试通过：
   - `pytest backend/open_webui/test/shared/test_seedance_shared_contracts.py`
   - `pytest backend/open_webui/test/shared/test_templates_integrity.py`
   - `pytest backend/open_webui/test/apps/webui/routers/test_seedance_tools_e2e.py`

## 发布流程

1. 代码修改后先执行同步：`python scripts/seedance/sync_template_imports.py`
2. 执行完整检查与测试。
3. 更新 `templates/versions/registry.json` 的 `updated_at` 与涉及组件版本。
4. 在 PR 描述中说明：变更组件、影响规则、回归测试结果。

## 回滚原则

- 优先回滚 `config/seedance_routing_registry.yaml` 对应 `rule_id`。
- 如行为错误来自工具实现，回滚对应 `templates/*_tool.py` 与同版本 `*.import.json`。
- 如行为错误来自技能/提示词约束，回滚对应 `templates/skills/*/SKILL.md` 与 `templates/prompts/*.txt`。

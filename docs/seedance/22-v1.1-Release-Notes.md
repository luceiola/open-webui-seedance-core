# Seedance Release Notes（v1.0 + v1.1）

发布日期：2026-04-30

## 1. 版本结论
- `v1.1` 开发收口完成，进入发布状态。
- `v1.0` 与 `v1.1` 双链路并存：
  - `v1.0`：素材包链路（`asset_package_id + @引用名`）
  - `v1.1`：媒体素材链路（`%相对路径`）

## 2. v1.0 回顾（已稳定）
### 2.1 交付内容
- 素材包后端能力：创建、查询、引用解析、生成提交、任务查询。
- Seedance 执行工具：`resolve -> generate -> task` 标准流程。
- 系统提示词与执行 Skill：约束结果格式、错误透传和任务状态回报。

### 2.2 核心价值
- 建立了可复用的视频生成编排主干。
- 形成了稳定的异步任务工作流（提交任务后按 `task_id` 查询）。

## 3. v1.1 新增（本次发布重点）
### 3.1 媒体素材独立链路（Media Assets）
- 新增路由：`/api/v1/media-assets/*`
- 支持能力：
  - 上传素材
  - 从聊天上传转换素材
  - 素材列表/详情
  - 临时访问 URL（TOS 预签名）

### 3.2 前端输入区能力增强
- `+` 菜单新增：
  - `Upload Assets`
  - `Upload Asset Folder`
  - `Reference Assets`
- 支持目录浏览与素材引用插入。
- 引用格式统一为：`%相对路径`。

### 3.3 生成工具侧升级
- `Seedance Material Package Tool v2` 扩展支持 `media-assets`。
- 在 media-assets 模式下支持：
  - `%引用` 解析
  - 缺失/歧义引用检测
  - 解析通过后再提交生成
- 任务返回结构统一包含状态与可读链接字段（例如 `video_url_markdown`）。

### 3.4 存储与访问
- 素材统一走 TOS。
- 返回访问链接采用预签名 URL。
- URL 有效期可通过环境变量配置（上限 7 天）。

## 4. 兼容性与行为说明
- 兼容 `v1.0` 素材包流程，不影响既有使用方式。
- `v1.1` 仅新增并行能力，不替换 `v1.0`。
- 生成任务仍采用异步模型：先返回任务标识，再查询终态。

## 5. 发布验收结论
- `v1.1` 主体开发完成。
- 核心交互链路已验证：上传与引用流程可用。
- 其余项按当前发布决策不再阻塞，版本收口。

## 6. 升级提示
- 若使用 `media-assets`，请确认：
  - 后端已挂载 `media-assets` 路由
  - TOS 配置完整
  - Tool/Skill/System Prompt 已在 OpenWebUI 控制台更新为最新版本

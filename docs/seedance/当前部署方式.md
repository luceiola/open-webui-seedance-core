当前部署方式可以概括为“**源码仓库 + 运行时补丁同步 + systemd 常驻**”：

1. 代码仓库  
`/home/baize/workspace/openwebui-seedance`（你维护的项目仓库）

2. 运行环境  
Conda 虚拟环境：`openwebui-seedance`  
OpenWebUI 实际运行在该环境的 `site-packages`

3. 配置方式  
核心密钥与参数放在：`config/ark.env`（通过 systemd `EnvironmentFile` 注入）

4. 启动方式  
`systemd` 服务 `openwebui-seedance.service` 常驻运行  
`ExecStart` 直接跑：`open-webui serve --host 0.0.0.0 --port 8801`

5. 发版更新流程（当前）  
`git pull` -> `scripts/sync_openwebui_patches.sh --python ...` -> `systemctl restart` -> 路由检查脚本验证

6. 当前特点  
优点是可控、可快速改；缺点是“补丁同步”步骤偏手工，容易漏同步（这正是 v1.1B 该优化的点）。

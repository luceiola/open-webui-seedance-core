# Seedance/HappyHorse 素材库 v1.2 接口契约（冻结草案）

发布日期：2026-04-30

> 说明：
> - 本文为 v1.2 的后端 API 冻结草案，用于前后端并行开发。
> - `v1.2` 发布前必须先完成 `v1.1.1` 与 `v1.1.2`：
>   - `23-v1.1.1-发布隔离方案.md`
>   - `24-v1.1.1-开发TODO.md`
>   - `25-v1.1.2-需求文档.md`
>   - `26-v1.1.2-开发TODO.md`
>   - `27-v1.1.2-接口契约.md`
> - 权限模型沿用 OpenWebUI（`read/write`）。
> - 旧 `media-assets` API 保留兼容，不在本契约列为主路径。

## 1. 总体约定

- Base Path：`/api/v1/material-library`
- 鉴权：`Authorization: Bearer <token>`
- 时间：Unix 秒级时间戳（`created_at/updated_at`）
- 错误结构：

```json
{
  "detail": "error message"
}
```

在业务失败场景（非 HTTP 异常）返回统一结构：

```json
{
  "ok": false,
  "error_code": "InvalidParameter",
  "error_message": "...",
  "request_id": "..."
}
```

## 3. 目录接口

### 3.1 获取目录树

`GET /api/v1/material-library/folders/tree`

Query：
- `include_stats`（bool，默认 true）

Response：

```json
{
  "root": {
    "id": "fld_root",
    "name": "ROOT",
    "children": [
      {
        "id": "fld_default",
        "name": "我的素材",
        "is_system": true,
        "children": [],
        "asset_count": 12,
        "subfolder_count": 3,
        "write_access": true
      }
    ]
  }
}
```

### 3.2 新建目录

`POST /api/v1/material-library/folders`

Request：

```json
{
  "parent_id": "fld_default",
  "name": "角色A"
}
```

Response：`Folder`

### 3.3 更新目录

`PATCH /api/v1/material-library/folders/{folder_id}`

Request：

```json
{
  "name": "角色A-新"
}
```

Response：`Folder`

### 3.4 删除目录（递归）

`DELETE /api/v1/material-library/folders/{folder_id}`

Query：
- `recursive=true`（必须）

Response：

```json
{
  "ok": true,
  "deleted": {
    "folder_count": 3,
    "asset_count": 18
  }
}
```

约束：
- `我的素材`（系统目录）不可删除，返回 400。

### 3.5 目录权限更新

`POST /api/v1/material-library/folders/{folder_id}/access/update`

Request：

```json
{
  "access_grants": [
    {"principal_type": "group", "principal_id": "grp_xxx", "permission": "read"},
    {"principal_type": "user", "principal_id": "user_yyy", "permission": "write"}
  ]
}
```

Response：

```json
{
  "ok": true,
  "folder_id": "fld_xxx",
  "access_grants": [
    {"id": "g1", "principal_type": "group", "principal_id": "grp_xxx", "permission": "read"},
    {"id": "g2", "principal_type": "user", "principal_id": "user_yyy", "permission": "write"}
  ]
}
```

### 3.6 获取目录网格数据（含面包屑与返回信息）

`GET /api/v1/material-library/folders/{folder_id}/entries`

Query：
- `q`（可选，关键字搜索）
- `limit`（默认 50）
- `offset`（默认 0）

Response：

```json
{
  "current_folder": {
    "id": "fld_2203",
    "name": "文件夹2203",
    "parent_id": "fld_default",
    "path": "我的素材/文件夹2203"
  },
  "breadcrumb": [
    {"id": "fld_default", "name": "我的素材"},
    {"id": "fld_2203", "name": "文件夹2203"}
  ],
  "can_go_back": true,
  "back_target_folder_id": "fld_default",
  "items": [
    {
      "entry_type": "folder",
      "id": "fld_sub",
      "name": "子目录",
      "media_type": null,
      "thumbnail_mode": "icon",
      "thumbnail_url": null,
      "thumbnail_fit": "cover",
      "updated_at": 1777000001
    },
    {
      "entry_type": "asset",
      "id": "asset_xxx",
      "name": "shot01.mp4",
      "media_type": "video",
      "thumbnail_mode": "image",
      "thumbnail_url": "https://...",
      "thumbnail_fit": "cover",
      "updated_at": 1777000002
    }
  ],
  "total": 2,
  "limit": 50,
  "offset": 0
}
```

渲染约束（前端必须遵循）：
- 目录与音频：`thumbnail_mode=icon`，按图标渲染。
- 图像与视频：`thumbnail_mode=image`，按 `thumbnail_fit=cover` 裁切填充网格。

## 4. 素材接口

### 4.1 上传素材

`POST /api/v1/material-library/assets/upload`

FormData：
- `folder_id`（required）
- `chat_id`（optional）
- `files[]`（required）

Response：

```json
{
  "ok": true,
  "uploaded": ["Asset"],
  "failed": [{"filename": "x.mov", "error": "File exceeds max size"}],
  "count": 2
}
```

### 4.2 从聊天上传转素材

`POST /api/v1/material-library/assets/from-upload`

Request：

```json
{
  "folder_id": "fld_xxx",
  "chat_id": "chat_xxx",
  "upload_ids": ["file_xxx"]
}
```

Response 同 `assets/upload`。

### 4.3 查询素材列表

`GET /api/v1/material-library/assets`

Query：
- `folder_id`（可选）
- `media_type`（image/video/audio）
- `status`（active/failed/...）
- `q`（搜索关键字）
- `limit`（默认 50）
- `offset`（默认 0）

Response：

```json
{
  "items": ["Asset"],
  "total": 123,
  "limit": 50,
  "offset": 0
}
```

### 4.4 查询素材详情

`GET /api/v1/material-library/assets/{asset_id}`

Response：`Asset`

### 4.5 获取素材预览 URL

`GET /api/v1/material-library/assets/{asset_id}/preview-url`

Query：
- `expires_in`（60~604800）

Response：

```json
{
  "asset_id": "asset_xxx",
  "url": "https://...",
  "expires_in": 3600
}
```

### 4.6 删除素材

`DELETE /api/v1/material-library/assets/{asset_id}`

Response：

```json
{
  "ok": true,
  "asset_id": "asset_xxx"
}
```

## 5. 封面接口

### 5.1 手动重试封面

`POST /api/v1/material-library/assets/{asset_id}/cover/retry`

Response：

```json
{
  "ok": true,
  "asset_id": "asset_xxx",
  "cover_status": "processing"
}
```

### 5.2 封面任务状态

`GET /api/v1/material-library/assets/{asset_id}/cover/status`

Response：

```json
{
  "asset_id": "asset_xxx",
  "cover_status": "ready",
  "retry_count": 2,
  "error": null
}
```

状态枚举：
- `processing`
- `ready`
- `failed`
- `placeholder`

## 6. `%引用` 接口

### 6.1 联想

`GET /api/v1/material-library/references/suggest`

Query：
- `q`（当前 `%` 后输入串）
- `limit`（默认 20）

Response：

```json
{
  "items": [
    {
      "asset_id": "asset_xxx",
      "relative_path": "角色A/walk.mp4",
      "media_type": "video",
      "preview_url": "https://...",
      "valid": true
    }
  ]
}
```

### 6.2 批量解析与有效性校验

`POST /api/v1/material-library/references/resolve`

Request：

```json
{
  "prompt": "让 %角色A/walk.mp4 在镜头中向前走"
}
```

Response：

```json
{
  "ok": true,
  "references": [
    {
      "raw": "%角色A/walk.mp4",
      "normalized": "角色A/walk.mp4",
      "asset_id": "asset_xxx",
      "media_type": "video",
      "valid": true,
      "reason": null
    }
  ],
  "missing": []
}
```

## 7. 生成前模型约束校验接口

`POST /api/v1/material-library/validate-for-generation`

Request：

```json
{
  "model": "happyhorse-1.0-r2v",
  "prompt": "... %角色A/walk.mp4 ...",
  "parameters": {
    "resolution": "720P",
    "ratio": "9:16",
    "duration": 5,
    "watermark": false,
    "seed": 123
  }
}
```

Response（通过）：

```json
{
  "ok": true,
  "resolved_assets": [
    {"asset_id": "asset_xxx", "media_type": "image", "relative_path": "角色A/a.png"}
  ]
}
```

Response（失败）：

```json
{
  "ok": false,
  "error_code": "ModelConstraintViolation",
  "error_message": "happyhorse-1.0-r2v only supports image references",
  "details": {
    "field": "references[0].media_type",
    "allowed": ["image"],
    "actual": "video"
  }
}
```

## 8. 状态接口

`GET /api/v1/material-library/tasks/{task_id}`

Response：

```json
{
  "task_id": "xxx",
  "status": "RUNNING",
  "error_code": null,
  "error_message": null,
  "request_id": null,
  "updated_at": 1777000000
}
```

状态必须覆盖：
- `PENDING`
- `RUNNING`
- `SUCCEEDED`
- `FAILED`
- `CANCELED`
- `UNKNOWN`

## 9. 与旧接口兼容策略

1. 旧接口 `/api/v1/media-assets/*` 保持可用。
2. 新界面优先调用 `/api/v1/material-library/*`。
3. 旧 `+` 引用入口通过 feature flag 控制显示。

## 10. 配置项（新增/沿用）

- `MATERIAL_LIBRARY_ENABLED=true|false`
- `MATERIAL_LIBRARY_LEGACY_ENTRY_VISIBLE=true|false`
- `MEDIA_ASSET_TOS_PRESIGN_EXPIRES_SECONDS`
- `MEDIA_ASSET_MAX_FILE_MB`
- `MATERIAL_LIBRARY_COVER_RETRY_MAX`（默认 3）
- `MATERIAL_LIBRARY_COVER_BACKOFF_BASE_MS`（默认 1000）

## 11. 验收重点

1. 接口字段稳定，不出现前后端字段名漂移。
2. 权限继承规则与 UI 展示一致。
3. `%引用` 可联想、可解析、可校验失效。
4. 图/视频/音频预览链路完整。
5. HappyHorse / Seedance 约束错误可直接定位字段。

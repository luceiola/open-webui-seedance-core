## 1. 目标

v1.2 目标是在 OpenWebUI 内完成“素材库”能力升级，支持：
- 独立入口（与工作空间、笔记同层）。
- 多级目录管理与权限继承。
- 图像/视频/音频统一管理。
- 对话输入 `%相对路径` 联想、插入与高亮。
- 管理页与对话页双端预览。
- 与 Seedance / HappyHorse 生成链路兼容。

## 2. 范围与边界

### 2.1 In Scope（v1.2）

1. 新增素材库独立页面与导航入口。
2. 新增目录树（根目录仅允许目录）。
3. 新增一级目录 ACL（`read/write`），子级继承。
4. 素材卡片网格视图（固定网格模式，3~8 列可调）。
5. `%相对路径` 联想与有效性高亮。
6. 图像/视频/音频预览浮层（管理页 + 对话页）。
7. 上传后封面生成（重试 + 占位图回退）。
8. 删除目录递归确认弹窗。
9. 旧 `+` 入口保留并支持开关隐藏。
10. 成功率指标：上传、插入、预览。

### 2.2 Out of Scope（v1.2 不做）

1. 所有权转移。
2. 跨组织素材同步。
3. 复杂视频剪辑/时间轴编辑。
4. 自动标签与语义检索（后续版本）。

## 3. 现状基线

当前已有能力：
- 后端 `media-assets` 路由：上传、列表、详情、临时 URL。
- 对话输入已有素材引用入口与 `%相对路径` 插入能力。
- TOS 预签名 URL 与过期时长可配置。
- Seedance / HappyHorse / GPT-Image2 工具链已基于素材引用能力扩展。

当前主要不足：
- 素材资产为“平铺记录”，缺少目录实体与权限继承。
- 对话与管理界面引用交互不统一，缺少高亮/失效反馈。
- 缺少素材库独立入口与完整管理操作（递归删除、目录 ACL）。

## 4. 核心设计

### 4.1 信息架构

顶层结构：
- 素材库（独立一级导航）
  - 根目录（系统，不可删，不存放媒体）
    - 一级目录（可设权限）
      - 任意级子目录（继承权限）
        - 素材（image/video/audio）

关键规则：
1. 根目录只放目录，不允许直接上传媒体。
2. 新用户默认创建 `我的素材` 一级目录（不可删除）。
3. 权限仅在一级目录配置，后代继承。
4. 不支持所有权转移。

### 4.2 权限模型

权限模型复用 OpenWebUI 现有 Access Grants：
- `permission`: `read` / `write`
- `principal_type`: `user` / `group`
- `principal_id`: 具体 user/group id（可含 `*`）

权限判定：
1. 用户为资源 owner => write。
2. 命中 `write` grant => write。
3. 命中 `read` grant => read。
4. 否则无权限。

继承逻辑：
- 一级目录可设置 grant。
- 子目录、素材不存独立 grant 字段，读取时从最近上级目录继承。

### 4.3 媒体类型与模型校验

素材库层（存储层）允许：
- `image`、`video`、`audio`

生成提交层（模型层）执行二次校验：
- HappyHorse：只允许图像参考（按官方接口约束）。
- Seedance：允许图像/视频/音频（按当前工具能力与模型约束）。

原则：
- 素材库不误拦截。
- 生成时精确拦截并返回字段级错误。

### 4.4 网格视图

交互要求：
1. 仅保留网格模式（不提供列表切换）。
2. 网格风格参照 Notes Grid。
3. 列数支持动态调整：最少 3 列，最多 8 列。
4. 支持搜索与筛选。
5. 网格上方必须显示目录导航条（面包屑），格式示例：`素材库 > 我的素材 > 文件夹2203`。
6. 面包屑各级可点击跳转到对应层级目录。
7. 必须提供“返回上一层”按钮，点击后进入当前目录的 `parent`。

卡片字段：
- 标题：素材名
- 次信息：类型 + 规格（图像分辨率 / 视频时长+分辨率 / 音频时长）
- 时间：更新时间
- 操作：预览、复制 `%相对路径`、删除等
- 缩略图规则：
  - 目录：使用目录图标（可选展示目录封面，未命中时回退图标）。
  - 音频：使用音频图标。
  - 图像/视频：使用缩略图，按网格比例 `cover` 裁切填充（不留黑边）。

### 4.5 `%相对路径` 联想与高亮

输入区行为：
1. 输入 `%` 触发联想面板。
2. 支持键盘 `↑/↓/Enter/Esc`。
3. 选中插入标准 `%相对路径`。
4. 插入后进行有效性校验：
   - 有效：高亮正常样式。
   - 失效（删除/无权限）：告警样式。

补充：
- 联想项支持点击预览。
- 联想内容显示：图/视频缩略图+名称，音频图标+名称。

### 4.6 预览浮层

覆盖两处：
- 素材库管理页
- 对话输入区

能力要求：
1. 图片查看（缩放可后续增强）。
2. 视频播放（基础控件）。
3. 音频播放（进度、时长）。
4. 关闭方式统一：Esc / 遮罩 / 关闭按钮。
5. URL 过期自动刷新一次；失败再提示。

### 4.10 目录导航交互（新增明确）

1. 面包屑导航条固定在网格上方，随目录变化实时更新。
2. 点击面包屑中的任意层级，直接切换到对应目录并刷新网格。
3. 返回按钮优先级：
   - 当前目录有 `parent_id`：回到上一级目录；
   - 当前目录已是根下一级：返回到“我的素材”列表视角（不退出素材库页面）。
4. 面包屑与返回按钮在素材库管理页和对话侧素材选择浮层保持一致行为。

### 4.7 上传与封面

封面流程：
1. 上传成功后进入异步封面任务。
2. 默认重试 3 次（指数退避）。
3. 连续失败写入占位图状态。
4. 支持手动“重新生成封面”。

### 4.8 删除策略

目录删除为递归删除：
- 目录本身
- 全部子目录
- 全部素材

强制确认弹窗展示：
- 目录数量
- 素材数量
- 不可恢复提示

例外：`我的素材` 不可删除。

### 4.9 任务状态机

统一状态：
- `PENDING`
- `RUNNING`
- `SUCCEEDED`
- `FAILED`
- `CANCELED`
- `UNKNOWN`

终态定义：
- `SUCCEEDED` / `FAILED` / `CANCELED` / `UNKNOWN`

失败返回：
- `error_code`
- `error_message`
- `request_id`（若可提取）

## 5. 数据模型（v1.2 新增）

### 5.1 目录实体 `media_asset_folder`

建议字段：
- `id`
- `owner_id`
- `name`
- `parent_id`（根下一级目录的 parent 固定为 root）
- `path`
- `is_system`（默认目录）
- `is_deleted`
- `created_at`
- `updated_at`

### 5.2 素材实体扩展 `media_asset`

在现有基础上新增：
- `folder_id`
- `cover_key` / `cover_status` / `cover_error`
- `width` / `height` / `duration_ms`（按类型可空）
- `deleted_at`

### 5.3 权限实体

复用 `access_grant` 表，不新增权限表。
- `resource_type` 新增 `media_asset_folder`
- `resource_id` 对应目录 id

## 6. 前后端改造点

### 6.1 后端

1. 新增素材目录 API。
2. 新增目录 ACL 更新 API。
3. 改造素材上传 API（必须指定目录）。
4. 新增目录递归删除逻辑。
5. 新增封面异步任务与状态字段。
6. 新增 `%引用` 有效性批量校验接口。

### 6.2 前端

1. 新建素材库页面（独立路由）。
2. 新建目录树 + 网格区双栏布局。
3. 新建列数控制器（3~8）。
4. 新增 ACL 弹窗复用 AccessControl 组件。
5. 对话输入 `%` 联想高亮升级。
6. 新增预览浮层组件（可复用）。

## 7. 兼容与迁移

1. 旧 `media-assets` 平铺数据需要迁移到目录结构：
   - 统一归档到 `我的素材/迁移数据/`。
2. 旧 `+` 入口保留（可隐藏开关）。
3. `%相对路径` 保持不变，避免存量 prompt 失效。
4. 失效路径在输入框显式标红提醒。

## 8. 可观测性与指标

1. 上传成功率。
2. `%插入成功率`。
3. 预览成功率。

建议日志维度：
- user_id
- folder_id
- asset_id
- model
- task_id
- request_id

1. 素材库入口与页面可用。
2. 目录树、权限继承、不可删目录规则生效。
3. 网格视图稳定（3~8 列、搜索筛选、面包屑可点击、返回上一层可用）。
4. `%联想 + 高亮 + 失效提示`生效。
5. 图/视频/音频预览可用，过期自动刷新一次；网格缩略图规则生效（目录/音频图标，图视频 cover 裁切）。
6. 上传封面重试与占位回退生效。
7. 递归删除可用且有确认弹窗。
8. HappyHorse/Seedance 模型约束校验准确。
9. 上传/插入/预览三项成功率可观测。

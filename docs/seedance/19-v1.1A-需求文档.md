# Seedance v1.1A 需求文档（素材上传与引用体验）

## 1. 目标

在不影响 v1.0 素材包主链路的前提下，新增一条“独立素材资产（media assets）”链路，提升聊天框 `+` 菜单下的素材上传和引用体验。

v1.1A 聚焦：
- 素材包逻辑保持不变。
- 新增独立媒体素材上传/引用能力（不走素材包入包逻辑）。
- 素材统一落 TOS，路径与素材包隔离。

## 2. 范围

### 2.1 In Scope

- 后端新增 `media-assets` 路由与存储。
- 支持从聊天上传文件（upload_id）转素材资产。
- 聊天输入区 `+` 菜单新增：
  - `Upload Assets`（上传素材）
  - `Reference Assets`（引用素材，二级列表）
- 点击“引用素材”中的条目，自动向输入框插入引用标记。
- 引用符号使用 `%`（避免与现有 `@/#/$/:` 冲突）。

### 2.2 Out of Scope

- 素材预览、时间轴、可视化编排。
- 素材资产与素材包自动互转。
- 服务端 Docker 化（后续版本处理）。

## 3. 关键设计

### 3.1 双链路并存

- 链路 A（已有）：素材包 `material-packages/*`
- 链路 B（新增）：素材资产 `media-assets/*`

两条链路互不干扰，便于后续择一收敛。

### 3.2 存储策略

- 素材资产走 TOS-only。
- 默认前缀：`media-assets/`。
- 与素材包默认前缀 `material-packages/` 分开。

### 3.3 引用约定

- 新增素材资产引用符号：`%素材名`
- 设计原因：
  - `@` 已用于模型/mention 生态，冲突风险高。
  - `%` 在当前输入框命令触发中未占用。

## 4. 功能需求

### FR-1 素材资产上传

- 支持“先上传文件到 OpenWebUI Files，再转为 media asset”。
- 支持多文件批量转换。
- 仅接受图片/视频/音频；不支持的文件给出失败项。

### FR-2 素材资产列表与详情

- 可按用户列出素材资产（默认新到旧）。
- 可查询单资产详情。
- 可获取临时访问 URL。

### FR-3 聊天输入区引用

- `+` 菜单提供 `Reference Assets` 二级列表。
- 点击条目后插入 `%素材名` 到输入框。

## 5. 验收标准

- 后端 `media-assets` 路由可用，支持上传/从 upload 转换/列表/详情/URL。
- `+` 菜单出现 `Upload Assets` 与 `Reference Assets`。
- 从 `Reference Assets` 点击素材，可自动插入 `%素材名`。
- 不影响 v1.0 素材包、生成任务相关流程。

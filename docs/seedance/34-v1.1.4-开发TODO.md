# Seedance v1.1.4 开发 TODO（统一模板任务接入）

发布日期：2026-05-08

## 0. 冻结目标

- [ ] 任务页只读 `/api/v1/tasks/*`，不引入第二任务中心。
- [ ] 三套模板工具统一上报任务状态到同一任务记录。
- [ ] 视频任务默认自动归档，下载不依赖上游临时 URL。
- [ ] 明确分层：`skill/tool/prompt` 可替换，后端接口契约长期稳定。

## 1. P0 后端桥接层

- [ ] 新增任务桥接入口（`/api/v1/tasks/bridge/upsert` 或等价内部服务）。
- [ ] 入参校验：`task_id/provider/status/tool_name/skill_name`。
- [ ] 状态归一化：`submitted/queued/in_progress/success/error/...` -> 标准状态。
- [ ] 复用 `_touch_task_record` 写入统一任务记录。
- [ ] 对视频成功任务触发归档轮询。
- [ ] 幂等约束：同一任务重复 upsert 无副作用（建议键：`provider+provider_task_id`）。
- [ ] 统一错误结构：便于工具层重试与故障判定。
- [ ] 写入失败不阻断主流程：记录失败并可补偿重放。

## 2. P0 Provider 刷新分发

- [ ] `ark` 刷新路径保持现状。
- [ ] 增加 `happyhorse` 查询适配（DashScope task query）。
- [ ] `openai_image2` 采用工具主动上报为主，不强依赖远端轮询。

## 3. P0 HappyHorse 接入

- [ ] `generate_video_with_happyhorse`：提交成功后上报 `PENDING`。

- [ ] `wait_happyhorse_task`：终态时补齐 `video_url/error/request_id`。
- [ ] 验证 HappyHorse 任务在任务页可见并可下载。

## 4. P0 GPT-Image2 接入

- [ ] 任务创建时上报 `PENDING`。
- [ ] 工作线程开始时上报 `RUNNING`。

- [ ] 失败时上报 `FAILED + error_code/error_message`。
- [ ] 保留原 `.data-dev/gpt_image2_tasks` 双写开关（默认开启）。

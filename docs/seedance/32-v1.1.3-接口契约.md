# Seedance v1.1.3 接口契约（统一任务中心，字段级冻结）

发布日期：2026-05-08

> 说明：
> - 本文用于 `v1.1.3` 后端/前端并行开发，字段级冻结。
> - 本契约覆盖：Seedance、HappyHorse 以及后续统一任务接入方。
> - 任务面板前端仅允许使用本契约中的 `/api/v1/tasks/*`。

## 1. 总体约定

- Base Path：`/api/v1/tasks`
- 鉴权：`Authorization: Bearer <token>`
- 时间字段：Unix 秒级时间戳（UTC）
- 编码：`application/json; charset=utf-8`
- 默认排序：`created_at desc, id desc`

分页约定：

1. `offset >= 0`
2. `1 <= limit <= 200`
3. 默认 `limit=48`

统一错误结构：

```json
{
  "ok": false,
  "error_code": "Forbidden",
  "error_message": "No permission to delete this task",
  "request_id": "req_xxx"
}
```

HTTP 状态码建议：

1. `400` 参数非法
2. `401` 未登录/鉴权失败
3. `403` 无权限
4. `404` 任务不存在
5. `409` 状态冲突（如 `ArchiveNotReady`、`NotCancelable`）
6. `429` 频率限制
7. `500` 服务内部错误

## 4. API 详情（冻结）

### 4.1 列表查询

`GET /api/v1/tasks`

Query：

| 参数 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| `user_id` | string | 否 | - | 用户筛选 |
| `provider` | string | 否 | - | provider 筛选 |
| `skill_name` | string | 否 | - | skill 筛选 |
| `tool_name` | string | 否 | - | tool 筛选 |
| `status` | string | 否 | - | `TaskStatus` |
| `model` | string | 否 | - | 模型筛选 |
| `include_deleted` | bool | 否 | `false` | 是否包含软删除 |
| `offset` | int | 否 | `0` | 偏移 |
| `limit` | int | 否 | `48` | 页大小，最大 200 |

Response：`TaskListResponse`

规则：

1. 默认不返回软删除任务。
2. 排序固定 `created_at desc, id desc`。
3. 列表中的 `can_delete/can_cancel` 按当前用户权限实时计算。

### 4.2 查询来源筛选项

`GET /api/v1/tasks/providers`

Query（可选）：

| 参数 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| `user_id` | string | 否 | - | 仅统计指定用户任务 |
| `include_deleted` | bool | 否 | `false` | 是否包含软删除任务 |

Response：

```json
{
  "providers": ["ark", "happyhorse"]
}
```

### 4.3 查询用户筛选项

`GET /api/v1/tasks/users`

Response：

```json
{
  "users": [
    {"user_id": "user_1", "user_name": "alice"},
    {"user_id": "user_2", "user_name": "bob"}
  ]
}
```

### 4.4 查询任务详情

`GET /api/v1/tasks/{task_id}`

Response：`TaskDetailResponse`

规则：

1. 非存在任务返回 `404`。
2. 软删除任务默认返回 `404`。
3. 该接口不返回 provider 原始敏感密钥字段。

### 4.5 查询预览信息

`GET /api/v1/tasks/{task_id}/preview`

Response：`TaskPreviewResponse`

### 4.6 下载任务产物

`GET /api/v1/tasks/{task_id}/download`

规则：

1. 仅允许下载归档成功的视频主产物。
2. 未归档成功返回 `409`，`error_code=ArchiveNotReady`。
3. 以文件流返回，必须带 `Content-Disposition: attachment`。

### 4.7 软删除任务

`DELETE /api/v1/tasks/{task_id}`

Query：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `delete_reason` | string | 否 | 最长 200 字符 |

Response：

```json
{
  "ok": true,
  "task_id": "tsk_xxx",
  "deleted_at": 1777000400
}
```

规则：仅创建者或管理员可执行。

### 4.8 归档重试

`POST /api/v1/tasks/{task_id}/archive/retry`

Response：

```json
{
  "ok": true,
  "task_id": "tsk_xxx",
  "archive_status": "PENDING"
}
```

规则：仅创建者或管理员可执行。

### 4.9 取消任务

`POST /api/v1/tasks/{task_id}/cancel`

Response：

```json
{
  "ok": true,
  "task_id": "tsk_xxx",
  "status": "CANCELED"
}
```

规则：

1. 仅创建者或管理员可执行。
2. 已终态任务返回 `409`，`error_code=NotCancelable`。
3. provider 不支持取消时返回 `409`，`error_code=ProviderNotCancelable`。

## 5. 存储 Schema（冻结）

### 5.1 tasks（逻辑结构）

```sql
id                 TEXT PRIMARY KEY
provider           TEXT NOT NULL
provider_task_id   TEXT NOT NULL
tool_name          TEXT NOT NULL
skill_name         TEXT NULL
user_id            TEXT NOT NULL
user_name          TEXT NOT NULL
chat_id            TEXT NULL
model              TEXT NULL
status             TEXT NOT NULL
archive_status     TEXT NOT NULL
progress           REAL NULL
error_code         TEXT NULL
error_message      TEXT NULL
request_id         TEXT NULL
deleted_at         BIGINT NULL
deleted_by         TEXT NULL
created_at         BIGINT NOT NULL
updated_at         BIGINT NOT NULL
finished_at        BIGINT NULL
```

约束与索引：

1. `UNIQUE(provider, provider_task_id)`
2. `INDEX idx_tasks_created(created_at DESC)`
3. `INDEX idx_tasks_user_created(user_id, created_at DESC)`
4. `INDEX idx_tasks_status_created(status, created_at DESC)`
5. `INDEX idx_tasks_deleted(deleted_at)`

### 5.2 task_artifacts（逻辑结构）

```sql
id                TEXT PRIMARY KEY
task_id           TEXT NOT NULL
artifact_type     TEXT NOT NULL
storage_backend   TEXT NOT NULL
storage_path      TEXT NOT NULL
mime_type         TEXT NULL
bytes             BIGINT NULL
width             INT NULL
height            INT NULL
duration_ms       BIGINT NULL
preview_ready     BOOLEAN NOT NULL DEFAULT FALSE
download_ready    BOOLEAN NOT NULL DEFAULT FALSE
created_at        BIGINT NOT NULL
```

约束与索引：

1. `FOREIGN KEY(task_id) REFERENCES tasks(id)`
2. `INDEX idx_artifacts_task(task_id)`
3. `UNIQUE(task_id, artifact_type, storage_path)`

## 6. 切流与回滚（冻结）

切流准入门槛：

1. 双写稳定运行 >= 24 小时。
2. 双写失败率 < 0.1%。
3. 新旧抽样字段一致率 >= 99.9%。
4. 回填覆盖率 >= 99.9%。
5. `GET /api/v1/tasks` P95 <= 500ms（`limit=48`）。
6. 已归档任务下载成功率 >= 99.0%。

强制回滚触发：

1. 5xx 比例连续 10 分钟 > 1.0%。
2. 双写失败率连续 10 分钟 > 1.0%。
3. 下载成功率连续 10 分钟 < 97.0%。
4. 出现高危越权且 30 分钟内无法修复。

## 7. 存储治理（冻结）

默认配置：

1. `TASK_SOFT_DELETE_RETENTION_DAYS=7`
2. `TASK_ARCHIVE_RETENTION_DAYS=90`
3. `TASK_ARCHIVE_MAX_RETRIES=3`
4. `TASK_ARCHIVE_MAX_FILE_MB=2048`
5. `TASK_ARCHIVE_USER_QUOTA_GB=200`

水位告警与策略：

1. `70%`：warning
2. `85%`：critical
3. `90%`：保护模式（归档限流，保下载优先）

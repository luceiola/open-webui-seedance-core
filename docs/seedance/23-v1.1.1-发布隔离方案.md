# Seedance v1.1.1 发布隔离方案（v1.1.2 / v1.2 前置）

发布日期：2026-05-07

## 1. 版本定位

1. `v1.1.1` 是 `v1.1.2` / `v1.2` 前的稳定性版本，不引入业务功能。
2. 目标是把 `8801`（prod）与 `8802`（dev）从“仅端口隔离”升级为“代码 + 配置 + 数据 + 发布流程”全隔离。
3. `v1.1.2` 与 `v1.2` 开发与上线以 `v1.1.1` 验收通过为前置条件。

## 2. 交付范围

1. 代码隔离：prod/dev 使用不同代码目录（或不同 worktree）。
2. 进程隔离：prod/dev 使用不同 systemd 服务实例。
3. 配置隔离：prod/dev 使用独立 `ENV_FILE` 与 `DATA_DIR`。
4. 存储隔离：prod/dev 使用不同 TOS 前缀（建议 `*-prod` / `*-dev`）。
5. 发布隔离：prod 仅通过“指定 commit/tag 发布”更新，不跟随开发目录实时变化。
6. 回滚隔离：prod 支持一键回滚到上一个 release commit/tag。

## 3. 实例约定（建议）

1. prod：
   - 端口：`8801`
   - 目录：`<repo-prod>`
   - env：`config/ark.env`
   - data：`.data-prod`
2. dev：
   - 端口：`8802`
   - 目录：`<repo-dev>`
   - env：`config/ark.dev.env`
   - data：`.data-dev`
3. TOS：
   - prod：`TOS_PREFIX=material-packages-prod`
   - dev：`TOS_PREFIX=material-packages-dev`
   - 如使用媒体专用前缀，同步区分 `MEDIA_ASSET_TOS_PREFIX`。

## 4. 发布流程（v1.1.1 后固定）

1. 开发只在 `<repo-dev>` 完成联调与验收。
2. 选定发布 commit/tag（例如 `release/v1.1.1`）。
3. 在 `<repo-prod>` 执行：
   - `git fetch --all --tags`
   - `git checkout <release-tag-or-commit>`
4. 重启 prod 服务（`8801`）并执行健康检查。
5. 验证通过后记录发布单（版本号、commit、发布时间、操作者）。

## 5. 回滚流程（必须具备）

1. 保留“上一稳定版本”tag（例如 `release/v1.1.0`）。
2. 异常时在 `<repo-prod>` 执行：
   - `git checkout <previous-release-tag>`
3. 重启 prod 服务并复测关键路径。
4. 回滚完成后登记故障与原因。

## 6. 验收标准（DoD）

1. 在 `8802` 改动并重启 dev，不影响 `8801` 当前行为。
2. 仅重启 `8801` 时，不会自动带入 dev 未发布改动。
3. 仅当 prod 目录更新到新 commit/tag 并重启后，`8801` 才生效新版本。
4. prod/dev 数据目录与云端素材前缀可确认独立。
5. 发布与回滚各完成至少一次演练并留存记录。

## 7. 与后续版本的关系

1. `v1.1.2` 与 `v1.2` 的功能开发可以在 dev 继续推进。
2. `v1.1.2` 与 `v1.2` 的正式发布必须沿用本方案的发布与回滚流程。
3. 文档顺序固定为：`v1.1` -> `v1.1.1` -> `v1.1.2` -> `v1.2`。

## 8. 已落地脚本（2026-05-07）

1. 开发部署脚本：`scripts/seedance/deploy_dev.sh`
2. 生产按 ref 发布脚本：`scripts/seedance/deploy_prod_ref.sh`
3. 配置隔离检查脚本：`scripts/seedance/check_env_isolation.sh`
4. systemd 示例：
   - `scripts/seedance/systemd/openwebui-seedance-prod.service.example`
   - `scripts/seedance/systemd/openwebui-seedance-dev.service.example`
5. 配置模板：
   - `config/ark.env.example`
   - `config/ark.dev.env.example`

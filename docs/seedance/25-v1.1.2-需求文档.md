# Seedance v1.1.2 需求文档（任务中心）

发布日期：2026-05-07

## 1. 版本定位

1. `v1.1.2` 目标是补齐“任务中心”能力，不引入 `v1.2` 的素材库目录/ACL 重构。
2. 核心价值：解决“上游临时 URL 过期导致任务结果不可下载”的问题。
3. 版本顺序：`v1.1` -> `v1.1.1` -> `v1.1.2` -> `v1.2`。

## 2. 背景与问题

当前任务查询结果依赖上游临时地址（预签名 URL），存在过期问题：

1. 任务成功后一段时间后，用户无法再次下载历史视频。
2. 历史任务缺少统一可视化面板，跨用户排查困难。
3. 前端缺少统一的“任务缩略图 + 预览 + 下载”闭环体验。

## 3. 目标

1. 新增一级导航入口 `[任务]`，与“笔记 / 工作空间”同级。
2. 所有用户可访问任务中心并查看全量任务列表。
3. 任务成功后由后端自动归档结果到服务器存储。
4. 用户下载始终走“平台归档文件”，不依赖上游临时 URL。
5. 成功任务展示缩略图，支持浮层预览与“下载到本地”。
6. 任务面板采用响应式网格展示，统一卡片比例与下载交互。

## 4. 范围

### 4.1 In Scope（v1.1.2）

1. 新增任务中心页面（全局入口）。
2. 任务列表与筛选：
   - 用户筛选（`user_id`）
   - 状态筛选（`PENDING/RUNNING/SUCCEEDED/FAILED/CANCELED/UNKNOWN`）
   - 模型筛选（`model`）
   - 时间排序（默认按 `created_at desc`）
3. 成功任务卡片展示视频缩略图（失败时占位图）。
4. 点击缩略图打开浮层，支持视频播放。
5. 浮层提供 `下载到本地` 图标；若有权限则同时展示删除图标。
6. 后端自动归档：
   - 任务成功后自动拉取视频并存储到平台侧。
   - 归档状态可追踪（成功/失败/重试）。
7. 删除任务：
   - 权限：任务创建者或管理员。
   - 逻辑：软删除。
   - 保留期：可配置，默认建议 7 天。
8. 任务面板样式与交互：
   - 网格模式，列数随屏幕宽度自适应在 3-8 列之间。
   - 每个任务 cell 固定宽高比 `3:4`。
   - 每个 cell 展示下载图标（点击即下载到本地）。
   - 浮层展示下载图标；若有删除权限则额外展示删除图标。

### 4.2 Out of Scope（v1.1.2 不做）

1. 素材库目录树、ACL 继承（`v1.2` 范围）。
2. 自动标签与语义检索。
3. 复杂视频编辑（剪辑、拼接、时间轴）。
4. 批量导出。

## 5. 关键规则（冻结）

1. 可见性：全员可见全量任务。
2. 删除权限：仅创建者或管理员可删。
3. 下载语义：
   - 用户侧下载动作只有“下载到本地”。
   - 后端归档为自动流程，不在前端暴露“归档”按钮。
4. 下载源必须是平台归档文件，不直接返回上游临时 URL。
5. 上游 URL 过期不应影响历史任务下载（若已归档成功）。

## 6. 状态机

任务主状态（对齐生成态）：

1. `PENDING`
2. `RUNNING`
3. `SUCCEEDED`
4. `FAILED`
5. `CANCELED`
6. `UNKNOWN`

归档子状态（新增）：

1. `NOT_REQUIRED`（非成功任务）
2. `PENDING`
3. `RUNNING`
4. `SUCCEEDED`
5. `FAILED`

## 7. 数据与展示字段

任务列表最小字段：

1. `task_id`
2. `user_id`
3. `user_name`
4. `model`
5. `status`
6. `archive_status`
7. `created_at`
8. `updated_at`
9. `thumbnail_url`（可空）
10. `download_ready`（bool）
11. `can_delete`（bool，控制浮层删除图标显隐）
12. `error_code` / `error_message` / `request_id`

## 8. 交互要求

1. 任务页默认分页加载，避免一次性拉取全部任务。
2. 状态变化支持手动刷新。
3. 失败任务显示结构化错误字段。
4. 删除动作必须二次确认，并提示“软删除保留期”。
5. 任务网格为响应式 3-8 列，卡片统一 `3:4`。
6. 列表卡片与浮层都提供下载入口。
7. 浮层删除图标仅在“创建者或管理员”身份下可见。

## 9. 风险与缓解

1. 全量任务可见带来隐私风险：
   - 缓解：默认列表仅展示 prompt 摘要，详情按需展开。
2. 自动归档带来存储增长：
   - 缓解：软删除 + 定期清理 + 保留期策略。
3. 上游波动导致归档失败：
   - 缓解：归档重试机制（指数退避 + 最大重试次数）。

## 10. 验收标准（DoD）

1. 普通用户可进入 `[任务]` 页面并查看全量任务。
2. 成功任务可展示缩略图、浮层预览、下载到本地。
3. 历史任务在上游 URL 过期后，仍可下载（已归档任务）。
4. 删除权限符合“创建者或管理员”规则。
5. 软删除保留期机制生效，并可审计删除行为。
6. 任务页满足响应式网格（3-8 列）与卡片比例（3:4）要求。
7. 列表卡片与浮层均可下载；浮层删除图标权限正确。

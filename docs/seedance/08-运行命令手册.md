# 运行命令手册（v1.1.1）

适用版本：`v1.1.1`  
核心原则：`dev(8802)` 与 `prod(8801)` 不仅端口隔离，还要代码目录隔离。

## 1. 目录与实例约定

建议使用两个目录（或两个 worktree）：

- 开发目录：`<repo-dev>`（只用于开发与联调，默认端口 `8802`）
- 生产目录：`<repo-prod>`（只用于发布与回滚，默认端口 `8801`）

数据目录约定：

- dev：`.data-dev`
- prod：`.data-prod`

## 2. 首次准备

复制配置文件（在各自目录执行）：

```bash
cp config/ark.env.example config/ark.env
cp config/ark.dev.env.example config/ark.dev.env
```

建议检查 prod/dev 配置是否隔离：

```bash
bash scripts/seedance/check_env_isolation.sh
```

说明：

- 该脚本会检查 `TOS_PREFIX` 是否区分 prod/dev。
- 若使用 `MEDIA_ASSET_TOS_PREFIX`，也会检查是否区分。

## 3. 启动方式

dev（8802）：

```bash
conda activate openwebui-seedance-core
cd <repo-dev>
bash scripts/seedance/run_openwebui_dev.sh
```

prod（8801）：

```bash
conda activate openwebui-seedance-core
cd <repo-prod>
bash scripts/seedance/run_openwebui_prod.sh
```

实例状态检查：

```bash
bash scripts/seedance/instance_status.sh
```

## 4. 开发部署（只用于 dev）

在 `<repo-dev>` 执行：

```bash
bash scripts/seedance/deploy_dev.sh --restart-systemd openwebui-seedance-dev
```

若不使用 systemd：

```bash
bash scripts/seedance/deploy_dev.sh --skip-healthcheck
HOST=0.0.0.0 PORT=8802 ENV_FILE=<repo-dev>/config/ark.dev.env DATA_DIR=<repo-dev>/.data-dev bash scripts/seedance/run_openwebui.sh
```

## 5. 生产发布（只用于 prod）

发布必须指定 tag/commit，在 `<repo-prod>` 执行：

```bash
bash scripts/seedance/deploy_prod_ref.sh --ref release/v1.1.1 --restart-systemd openwebui-seedance-prod
```

说明：

- 脚本会 `git fetch --tags` 后 `git checkout <ref>`。
- 发布完成会打印一条“回滚命令提示”。

## 6. 生产回滚（只用于 prod）

```bash
bash scripts/seedance/deploy_prod_ref.sh --ref <previous-release-tag-or-commit> --restart-systemd openwebui-seedance-prod
```

## 7. 健康检查与路由验证

健康检查：

```bash
bash scripts/seedance/healthcheck.sh http://127.0.0.1:8801
bash scripts/seedance/healthcheck.sh http://127.0.0.1:8802
```

素材包路由检查：

```bash
curl --noproxy '*' -fsS http://127.0.0.1:8801/openapi.json \
| jq -r '.paths | keys[]' \
| grep '/api/v1/material-packages'
```

素材资产路由检查：

```bash
curl --noproxy '*' -fsS http://127.0.0.1:8802/openapi.json \
| jq -r '.paths | keys[]' \
| grep '/api/v1/media-assets'
```

## 8. systemd 示例文件

仓库提供示例：

- `scripts/seedance/systemd/openwebui-seedance-prod.service.example`
- `scripts/seedance/systemd/openwebui-seedance-dev.service.example`

使用时请替换：

- `User`
- `WorkingDirectory`
- `EnvironmentFile`
- `ExecStart` 绝对路径

## 9. 常见问题

1. `greenlet` 缺失  
- 执行：`pip install greenlet`

2. `material-packages` 路由不存在  
- 执行：`bash scripts/seedance/check_material_routes.sh`
- 确认当前服务确实来自对应目录（prod 或 dev）。

3. dev 改动“意外进了”prod  
- 检查是否误在 prod 目录执行了 `deploy_dev.sh` 或 `git checkout`。
- 检查 systemd 的 `WorkingDirectory` 是否指向了错误目录。

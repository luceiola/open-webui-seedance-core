# 运行命令手册（v1.1B）

适用仓库：`/Users/lucas/Documents/open-webui-seedance-core`  
实例约定：
- 生产实例：`8801`
- 开发实例：`8802`

## 1. 开发环境启动（推荐）

```bash
conda activate openwebui-seedance-core
cd /Users/lucas/Documents/open-webui-seedance-core
bash scripts/seedance/run_openwebui_prod.sh
```

说明：
- 该脚本会先执行 preflight（检查 `open_webui`、`greenlet`、前端目录链接）。
- 若存在 `config/ark.env`，会自动加载。
- 生产实例默认 `DATA_DIR=.data-prod`，开发实例默认 `DATA_DIR=.data-dev`，彼此隔离。

## 2. 预检查（P0）

```bash

cd /Users/lucas/Documents/open-webui-seedance-core
```

## 5. 一键部署流程（P1）

### 5.1 非 systemd（只做更新+检查）

```bash
conda activate openwebui-seedance-core
cd /Users/lucas/Documents/open-webui-seedance-core
bash scripts/seedance/deploy.sh --skip-healthcheck
```

然后手动启动：

```bash
HOST=0.0.0.0 PORT=8801 bash scripts/seedance/run_openwebui.sh
```

### 5.2 systemd（更新+重启+检查）

```bash
conda activate openwebui-seedance-core
cd /Users/lucas/Documents/open-webui-seedance-core
bash scripts/seedance/deploy.sh --restart-systemd openwebui-seedance
```

## 6. OpenAPI 验证（关键）

检查素材包路由：

```bash
curl --noproxy '*' -fsS http://127.0.0.1:8801/openapi.json \
| jq -r '.paths | keys[]' \
| grep '/api/v1/material-packages'
```

检查素材资产路由（v1.1A）：

```bash
curl --noproxy '*' -fsS http://127.0.0.1:8802/openapi.json \
| jq -r '.paths | keys[]' \
| grep '/api/v1/media-assets'
```

如果你只开了 8801，把命令中的 `8802` 改为 `8801`。

## 7. 常见问题

1. `/` 返回 `{"detail":"Not Found"}`
- 先执行：
  - `bash scripts/seedance/preflight.sh --auto-fix`
- 再重启服务。

2. 启动时报 `No module named 'greenlet'`
- 执行：
  - `pip install greenlet`

3. Material routes 不存在
- 确认当前运行的是 `openwebui-seedance-core` 对应环境与代码。
- 重跑：
  - `bash scripts/seedance/check_material_routes.sh`
